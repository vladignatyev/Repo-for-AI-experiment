Concept: Android finance tool for mass-market investors

What we are building, for whom, and why it wins.

Target user: mass investing segment (new to investing or “busy professional”). They want quick, confident answers like “how much do I need to retire?” without a spreadsheet.

Core value: convert vague goals → a number + a plan (next actions).

Key principles:

  • One-screen inputs, immediate results, sensible defaults.
  • Guardrails: show assumptions (SWR, inflation) and warn about unrealistic inputs.
  • Explainability: “where the number comes from” in plain language.
  • Privacy-first: offline calculators; optional cloud sync later.

MVP modules:

  1. Compound growth (principal + monthly contribution)
  2. Retirement target (monthly spending + SWR)
  3. FIRE timeline (assets + savings rate + real return)
  4. Passive income (SWR vs yield)

Monetization (later): premium scenarios (multiple portfolios), export PDF, tax presets by country, Monte Carlo, coaching prompts.

Android implementation sketch: Kotlin + Jetpack Compose, local persistence (Room/DataStore), charts (MPAndroidChart or Compose Canvas), optional onboarding wizard.

Prototype notes
  • This HTML is intentionally “Android-like”: big tap targets, cards, simple nav.
  • Math is simplified; production should include scenario ranges and country-specific caveats.
